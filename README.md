# 👸 Мульт-няня (Mult-Nanny)

Інтерактивний додаток для дітей 5–8 років, який може розмовляти, розповідати казки та грати в ігри за допомогою AI (OpenAI GPT-4).

## 🚀 Як почати користуватись (Локально)

### 1. Підготовка
Вам знадобляться встановлені:
- **Node.js** (для основного сервера)
- **Python 3.11+** (для TTS сервера)

### 2. Встановлення залежностей
```bash
npm install
```

### 3. Налаштування
Вкажіть ваш `OPENAI_API_KEY` у файлі `start.sh` або як змінну оточення.

### 4. Запуск
```bash
./start.sh
```
Або окремо:
```bash
npm run start
```
Додаток буде доступний за адресою: `http://localhost:3000`

## 🌐 Деплой на GitHub Pages

Цей проект налаштований так, що він може працювати на **GitHub Pages** навіть без власного бекенду.

### 1. Перший деплой
1. Створіть новий репозиторій на GitHub.
2. Виконайте команди:
   ```bash
   git init
   git add .
   git commit -m "Initial commit"
   git branch -M main
   git remote add origin https://github.com/ВАШ_ЛОГІН/ВАШ_РЕПО.git
   git push -u origin main
   ```
3. GitHub Actions автоматично збере проект та розгорне його на гілку `gh-pages`.

### 2. Налаштування на GitHub
- Зайдіть у **Settings -> Pages** вашого репозиторію.
- Переконайтеся, що джерелом деплою є гілка `gh-pages`.

### 3. Використання на GitHub Pages
Оскільки на GitHub Pages немає сервера Node.js, вам потрібно:
1. Відкрити розгорнутий сайт (наприклад, `username.github.io/nany/`).
2. Натиснути кнопку **"Налаштування"** зверху.
3. Вставити ваш **OpenAI API Key**.
4. Ключ збережеться у вашому браузері (LocalStorage) і Няня зможе спілкуватись напряму.

## ⚠️ Безпека
- **Ніколи** не завантажуйте свій `OPENAI_API_KEY` у публічні репозиторії у відкритому вигляді (наприклад, у файлі `start.sh`).
- Файл `.gitignore` вже налаштований на ігнорування системних файлів, але будьте уважні.

## 🛠 Технології
- **Frontend**: Vanilla JS, HTML5, CSS3.
- **Backend**: Node.js (Express), Python (FastAPI/Uvicorn).
- **AI**: OpenAI Chat Completions API.
- **TTS**: Локальний Python сервер або вбудований Voice Synthesis браузера (як fallback).
